## 2024-05-24 - [HIGH] Fix weak hashing algorithms
**Vulnerability:** MD5 and SHA-1 algorithms were being used for cryptographic hashes (e.g. `hashlib.md5` and `hashlib.sha1`) in several modules (`functions/cache.py`, `functions/telegram_bot.py`, `functions/user_storage.py`).
**Learning:** Usage of insecure algorithms like MD5 and SHA1 poses a security risk due to known vulnerabilities and collision weaknesses. Even when used for non-critical identifiers, it is a bad practice and violates security policies.
**Prevention:** Establish and strictly enforce a coding standard that mandates the use of modern, secure hash functions (such as SHA-256) for all hashing purposes. Here, we implemented a centralized `stable_hash` utility (using SHA-256) to ensure consistent and secure hashing across the codebase.
## 2024-04-01 - SSRF in Summarize Function
**Vulnerability:** The application fetches user-provided URLs in `functions/telegram_bot.py` via `httpx.AsyncClient` without any SSRF protection, and with `follow_redirects=True`, opening it up to accessing internal network resources.
**Learning:** Even when users provide links ostensibly to public news articles or tools, these links must be verified as safe (not pointing to internal IPs like localhost or metadata services) prior to making HTTP requests. The `follow_redirects` feature in HTTP clients can bypass initial URL checks by redirecting to a private IP, so redirects must also be validated or strictly controlled.
**Prevention:** Always validate all user-supplied URLs and all subsequent redirect targets against a strict list of allowed external IP ranges. Avoid relying solely on `httpx`'s automatic redirect following if redirect destinations are not validated.
