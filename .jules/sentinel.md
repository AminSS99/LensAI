## 2024-05-24 - [HIGH] Fix weak hashing algorithms
**Vulnerability:** MD5 and SHA-1 algorithms were being used for cryptographic hashes (e.g. `hashlib.md5` and `hashlib.sha1`) in several modules (`functions/cache.py`, `functions/telegram_bot.py`, `functions/user_storage.py`).
**Learning:** Usage of insecure algorithms like MD5 and SHA1 poses a security risk due to known vulnerabilities and collision weaknesses. Even when used for non-critical identifiers, it is a bad practice and violates security policies.
**Prevention:** Establish and strictly enforce a coding standard that mandates the use of modern, secure hash functions (such as SHA-256) for all hashing purposes. Here, we implemented a centralized `stable_hash` utility (using SHA-256) to ensure consistent and secure hashing across the codebase.
## 2025-02-28 - [SSRF via user URL summarization]
**Vulnerability:** The application fetches and summarizes arbitrary user-provided URLs in `summarize_url_callback` (`functions/telegram_bot.py`). If not validated, an attacker can input URLs resolving to internal IP addresses (e.g., `127.0.0.1`, `169.254.169.254`, `192.168.x.x`), potentially causing Server-Side Request Forgery (SSRF) and allowing them to scan local networks or fetch cloud metadata.
**Learning:** Functions that fetch external resources based on unvalidated user input are prime targets for SSRF. A common missing link is failing to validate the IP address that the hostname resolves to.
**Prevention:** Implement and use a validation function like `is_safe_url` to restrict allowed schemes (HTTP/HTTPS), resolve the hostname to an IP, and explicitly reject loopback, link-local, and private IP ranges before initiating any external network request.
